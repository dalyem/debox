# Debox Architecture

Debox is a **platform** that hosts many games, plus three shipped games (Phase
Cards, Spades, and Cheat/Bullshit). This document explains how the platform is
layered so that a new game is a *module*, not a rewrite.

## System overview

```
        ┌──────────────────────────┐         ┌──────────────────────────┐
        │   TV / shared display     │         │   Phone controllers       │
        │   /host/[roomId]          │         │   /play/[roomCode]         │
        │   (Clerk-authed host)     │         │   (anonymous guests)       │
        └────────────┬─────────────┘         └────────────┬─────────────┘
                     │  useQuery / useMutation (reactive websocket)        │
                     ▼                                                     ▼
        ┌─────────────────────────────────────────────────────────────────┐
        │                          CONVEX                                   │
        │                                                                   │
        │   rooms.ts  players.ts  gameplay.ts  events.ts  sessions.ts       │
        │   games.ts  crons.ts            (public + internal functions)     │
        │        │            │                                             │
        │        ▼            ▼                                             │
        │   lib/engine.ts  ── the ONLY bridge ──►  GameEngine (pure)        │
        │   lib/auth.ts  lib/guestTokens.ts  lib/roomCodes.ts               │
        │                                                                   │
        │   schema.ts: users rooms players games gameState moves events     │
        └─────────────────────────────────────────────────────────────────┘
                     ▲
                     │  imports the SAME pure modules (no duplication)
        ┌─────────────────────────────────────────────────────────────────┐
        │   src/lib  (isomorphic, dependency-free)                          │
        │   platform/ (rng, turn, types)   cards/ (deck, validate)          │
        │   games/ (registry, types, phase-cards/*)                         │
        └─────────────────────────────────────────────────────────────────┘
```

Two ideas make the whole thing work:

1. **Games are pure state machines.** A game never touches Convex, the network,
   or the clock. The platform owns all I/O and calls the game's pure reducers.
2. **The platform speaks one game-agnostic vocabulary** (rooms, players,
   sessions, events, turns). Adding a game adds a reducer + two views, nothing
   else.

> **Why `src/lib` is imported by Convex:** the platform engines and the card/game
> logic are pure and isomorphic, so they live once in `src/lib` and are imported
> by both the browser and Convex (via relative paths — the Convex bundler does
> not resolve the `@/` alias, so shared modules use relative imports). One source
> of truth, zero drift between client previews and server authority.

## Platform engines

Each "engine" is a focused, reusable responsibility. None of them know anything
about a specific game.

| Engine | Lives in | Responsibility |
| --- | --- | --- |
| **Room Engine** | `convex/rooms.ts` | Create rooms, lifecycle transitions, status, codes, dashboard queries |
| **Player Engine** | `convex/players.ts` + `src/lib/platform/avatars.ts` | Anonymous guests, seats, reconnect/heartbeat, roster |
| **Session Engine** | `convex/sessions.ts` + `convex/crons.ts` | Expiry, cleanup, sweeps; terminal rooms never reopen |
| **Realtime Engine** | Convex reactive queries + `src/components/platform/EventToaster.tsx` | Subscriptions + event-driven animation; no polling |
| **Turn Engine** | `src/lib/platform/turn.ts` | Pure turn order, direction, skips, eligibility |
| **Game Engine** | `convex/lib/engine.ts` | Loads/saves state, calls the game reducers, fans out events |
| **Event Engine** | `convex/lib/engine.ts` (`emit`/`recordEvents`) + `convex/events.ts` | Durable, ordered event feed for notifications/animations |

The **Game Engine bridge** (`convex/lib/engine.ts`) is the single seam between
the platform and any game:

```ts
launchGame(ctx, room)        // createGame + startGame → persist + emit
applyPlayerMove(ctx, …, move)// validate (server) → submitMove → persist + emit
resumeRound(ctx, room)       // round-summary pause → deal next round
publicView(ctx, room)        // TV projection (audience-safe)
privateView(ctx, room, pid)  // controller projection (only your hand)
```

## Game plugin system

A game implements one TypeScript interface, `GameEngine<State, Move, Config>`
(`src/lib/games/types.ts`). Every method is **pure** — given state + input it
returns new state + events, with randomness supplied as a numeric `seed`:

```ts
createGame()      joinGame()        startGame()
validateMove()    submitMove()      advanceTurn()      resume?()
endGame()         getPublicState()  getPrivateState()
serializeState()  deserializeState()
```

Games register themselves with the **registry** (`src/lib/games/registry.ts`):

```ts
// src/lib/games/index.ts — imported by both server and browser
registerGame(PhaseCardsEngine);
registerGame(SpadesEngine);
registerGame(CheatEngine);
// registerGame(TriviaEngine);   ← future
```

The **client** mirrors this with a parallel view registry
(`src/components/games/registry.ts`) mapping a `gameType` → `{ Tv, Controller }`
React components. The host (`/host/[roomId]`) and controller
(`/play/[roomCode]`) pages are fully game-agnostic: they own the platform chrome
(lobby, header, results, reactions, post-game choices) and delegate the *active
board* to the registered views, keyed by the room's `gameType`. Every game's
projections expose a small shared base (`status` / `isYourTurn` / `turnDeadline`)
so the pages can drive the turn timer, "your turn" banner and round pacing
without knowing the game.

Because the registry is the single source of truth:

- The dashboard's game picker (`games.list`) reads it.
- The Convex bridge resolves `room.gameType → engine` from it.
- The browser can run the *same* validators for instant, optimistic UI feedback
  (e.g. the lay-down builder previews validity before the server confirms it).

**Why pure reducers?** Server-authoritative (all validation in Convex),
deterministic + replayable (seed + move log reproduces any state),
trivially unit-testable (no mocks — see `*.test.ts`), and fully decoupled
(a new game can't break the room system because it can't reach it).

### Audience-safe projections

State is stored once, authoritatively, as a serialized blob. Clients never read
it directly — they read **projections**:

- `getPublicState(state, players)` → the TV view: counts, melds (face-up),
  whose turn, the discard top. **No hands.**
- `getPrivateState(state, playerId, players)` → one controller: *your* hand plus
  the public table. Other players' hands are physically absent from the payload.

This is enforced on the server, so "players cannot view other hands" is a
property of the wire format, not a UI nicety.

## Data model

`convex/schema.ts` (indexes in parentheses):

| Table | Purpose | Key fields |
| --- | --- | --- |
| `users` | Authenticated hosts (Clerk) | `clerkId` *(by_clerk)*, email, name |
| `rooms` | One game session | `hostClerkId`, `roomCode` *(by_code)*, `gameType`, `status`, `expiresAt` *(by_status_expiry)*, `eventCursor`, `moveCursor`, `result` |
| `players` | Anonymous guests | `roomId` *(by_room)*, `displayName`, `guestTokenHash` *(by_room_token)*, `seat`, `isActive` |
| `games` | Catalog mirror of the registry | `gameId` *(by_gameId)*, metadata, `enabled` |
| `gameState` | Authoritative serialized game state | `roomId` *(by_room)*, `state` (JSON string), `runtimeStatus`, `version`, `round` |
| `moves` | Append-only move log (audit/replay) | `roomId` *(by_room_seq)*, `seq`, `type`, `payload` |
| `events` | Ordered notification/animation feed | `roomId` *(by_room_seq)*, `seq`, `type`, `audience`, `payload` |

The game's full state lives as a string in `gameState.state`, so **adding a game
never migrates the schema**. `eventCursor`/`moveCursor` on the room give events
and moves a stable monotonic order without extra reads.

## Realtime model

Convex queries are reactive: any component that `useQuery`s a function
re-renders automatically when underlying data changes. There is **no polling**.

- **Lobby:** `players.list` + `rooms.summary` → players pop in as they join.
- **Table:** `gameplay.publicState` (TV) and `gameplay.privateState` (controller)
  recompute from `gameState` on every move.
- **Animations:** the `events` feed is an append-only, ordered log. The
  `EventToaster` subscribes, ignores history on first load, and animates only
  events newer than the last `seq` it has seen — so card draws, freezes, phase
  completions and "went out!" surface as transient toasts on every screen.
- **Pacing:** when a round ends, the bridge schedules
  `internal.gameplay.resumeAfterRound` via `ctx.scheduler.runAfter(...)`, so the
  TV shows the scoreboard for a beat before the next deal (the host can also
  "Deal now").

## Security model

- **Only hosts authenticate** (Clerk). `convex/auth.config.ts` validates Clerk
  JWTs; `requireHostRoom` gates every host-only mutation on
  `room.hostClerkId === identity.subject`.
- **Players are anonymous** but hold a **signed guest session token**
  (`convex/lib/guestTokens.ts`): ~244 bits of entropy, returned once on join. The
  server stores only a `SHA-256` hash **peppered with a server-side secret**
  (`GUEST_TOKEN_SECRET`). The pepper never leaves the server, so the hash can't
  be forged or reversed. Every player action re-derives the hash and looks the
  player up — the client never asserts its own identity.
- **All validation is server-side.** `gameplay.submitMove` re-runs
  `engine.validateMove` before applying. The client's optimistic preview is a
  convenience, never the source of truth.
- **Audience-safe projections** (above) keep hands and host controls off the
  wire for players.

## Session lifecycle

```
pending → lobby → active ⇄ paused → ended → closed
                    │                  │         ▲
                    └──── (idle) ──────┴── expired (terminal)
```

- A room is created in **lobby**; players join only while in lobby.
- **start** deals and moves to **active**; **pause/unpause** toggle **paused**.
- **end** finalizes standings → **ended**; after a grace period it auto-**closes**.
- Abandoned rooms auto-**expire**: each room arms a precise
  `scheduler.runAfter(expiry, expireIfIdle)` on create, and a 5-minute `sweep`
  cron is the backstop. Any activity (`touchRoom`) pushes `expiresAt` forward, so
  live games never expire.
- **closed** and **expired** are terminal. Revisits get
  *"This game session has ended."*

## The card engine & shipped games

**Card Engine** (`src/lib/cards/`) is a reusable, game-agnostic framework with
two decks:

- The **colored Debox deck** (`types.ts`, `deck.ts`, `validate.ts`): `createDeck`,
  `shuffleDeck`, `dealCards`, `drawCard`, `discardCard`, `recycleDiscard`, and
  validators `validateSet` / `validateRun` / `validateColorGroup` with full wild
  ("Shift") and skip ("Freeze") handling. 108 cards (1–12 in four colors ×2, 8
  Shifts, 4 Freezes). Backs **Phase Cards**.
- The **standard 52-card deck** (`standard.ts`): `Suit`/`Rank`/`StandardCard`,
  `createStandardDeck`, `shuffleStandard`, `dealHands` (fixed count) / `dealAll`
  (even split), `compareRank`, `compareSuit`, `sortStandardHand`, plus labels and
  the Cheat rank cycle. Backs **Spades** and **Cheat**.

Both decks shuffle through the same injected seeded RNG (`src/lib/platform/rng.ts`)
so deals are deterministic and replayable.

**Phase Cards** (`src/lib/games/phase-cards/`) — an original ten-phase climb (see
`phases.ts`); draw/lay-down/hit/freeze/discard with round scoring and win
detection.

**Spades** (`src/lib/games/spades/`) — partnership trick-taking for exactly four
(seats 0&2 vs 1&3). Bidding → 13 tricks (follow suit, spades trump, can't lead
spades until broken) → team scoring with bags + nil (pure, in `scoring.ts`).
First team to 500 wins. `round_over` between hands uses the platform's `resume`
pacing; stalled turns auto-bid/auto-play via `autoResolveTurn`.

**Cheat / Bullshit** (`src/lib/games/cheat/`) — 3–8 players. The whole deck is
dealt; the required rank advances each turn (A→2→…→K→loop). Players play cards
face-down with an auto-claim of the required rank (and may lie); anyone may call
"Cheat!". A challenge reveals the cards — the liar or the wrong challenger eats
the pile. First to empty their hand wins (a win only locks in once the final play
can no longer be challenged).

Every engine is covered by pure unit tests for rules, scoring, turn flow and
information hiding (`*.test.ts`).

### Quick reactions

Players can fling emoji (😂 😮 👀 🤔 💀) at the shared screen. This rides the
existing **event feed** — no schema change: `gameplay.sendReaction` (player-auth)
emits a `reaction` event, the host's `ReactionsLayer` animates it as a brief
floater, and the controller's `ReactionFab` sends it. The toaster ignores
reactions so they never clutter the notification stream.

## Design system

A deliberately non-corporate identity (`src/app/globals.css` + `src/lib/design/`):
deep neon "stage" background, chunky rounded **sticker surfaces** with depth, a
bold display face (Fredoka) over Inter, an animated neon wordmark, and 3D
console-style buttons. Tailwind CSS v4 tokens (`@theme`) expose the palette as
utilities; Framer Motion animates joins, deals, turn changes, the lay-down, round
summaries and the confetti victory screen.

## Scalability

- **Indexed, bounded reads.** Every query hits an index (`by_room`,
  `by_status_expiry`, `by_room_token`, …) and `take()`s bounded pages — no table
  scans, so cost is independent of total room count.
- **Reactivity, not polling.** Hundreds of rooms and thousands of phones each
  subscribe only to their own room's queries; Convex pushes minimal diffs.
- **Per-room state is one row.** The whole game is a single serialized
  `gameState` document, so a move is one read + one write + a few event inserts.
- **Self-cleaning.** Scheduled expiry + the sweep cron keep working sets small.
- **Codes recycle.** Room codes are unique only among *live* rooms, so the code
  space never exhausts.
